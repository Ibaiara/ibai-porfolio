## Proyectos destacados

### 🎮 IbaiRPG — Motor narrativo con IA

**RPG narrativo político–estratégico impulsado por IA**, centrado en la toma de decisiones y la gestión de consecuencias.

Este proyecto nace del interés por **entender y experimentar con nuevas formas de trabajar con IA generativa**, no solo desde el punto de vista del prompt, sino desde la **orquestación de flujos, el control del estado y la coherencia del sistema**.

El motor:
- utiliza **n8n** como orquestador de flujos  
- gestiona estado persistente  
- integra generación de texto, imagen y voz  
- separa claramente contenido, lógica y presentación  
- utiliza escenas estructuradas en lugar de branching libre  

El uso de un RPG es intencional: permite explorar estos conceptos de forma creativa y salir de la rutina de proyectos puramente empresariales, manteniendo un enfoque técnico sólido.

### 🧳 Gestión de viajes — Aplicación de escritorio y demo web

Aplicación desarrollada inicialmente como **app de escritorio con Electron** para la gestión de viajes de operarios y comerciales en un entorno empresarial real.

El proyecto aborda problemas reales como:
- desorganización de viajes  
- falta de trazabilidad  
- dificultad para visualizar desplazamientos y calendarios  
- necesidad de exportar datos para análisis (Power BI)  

Características destacadas:
- frontend con calendarios y mapas interactivos  
- backend en Node.js con persistencia local y backups  
- importación de datos desde Excel  
- API REST clara y modular  

Para el portfolio, el proyecto se presenta mediante:
- una **demo web** que reutiliza la lógica central del backend  
- (opcionalmente) el ejecutable original de Electron  

🔗 Demo web: https://travel-manager-web.onrender.com/  
🔗 Código fuente: https://github.com/Ibaiara/TravelManagementApp

## Enfoque técnico

En mis proyectos personales priorizo:

- arquitectura clara y comprensible  
- separación de responsabilidades  
- control explícito del estado  
- decisiones técnicas justificadas  
- uso pragmático de herramientas (sin sobreingeniería)  

No busco “hacerlo todo con la última tecnología”, sino **entender por qué una tecnología tiene sentido en un contexto concreto**.
